# GEO 课程端到端流程验证 - Product Requirement Document

## Overview
- **Summary**: 对微信小程序进行全面的端到端业务流程验证，确保页面之间的跳转、接口调用、数据传递能够完整串联，用户可以顺畅地完成核心业务流程。
- **Purpose**: 发现并修复页面跳转断裂、接口数据不一致、参数传递错误等问题，确保用户体验流畅。
- **Target Users**: 测试人员、产品经理、开发人员

## Goals
- 验证首页 → 课程列表 → 课程详情 → 课时播放 的完整浏览链路
- 验证首页 → 讲师列表 → 讲师详情 的讲师浏览链路
- 验证个人中心 → 各子页面 的菜单跳转链路
- 验证学习页面 → 继续学习 的学习链路
- 验证所有页面的 API 接口调用正常（useLocal=false 模式）
- 验证页面间参数传递（课程ID、课时ID等）正确
- 验证购买/微信小店流程可正常触发

## Non-Goals (Out of Scope)
- 不进行微信开发者工具中的真机交互测试（仅代码级验证）
- 不验证微信小店的真实支付流程
- 不验证登录注册的完整流程（需微信授权）
- 不进行性能测试和压力测试
- 不修改业务逻辑，仅修复流程断裂问题

## Background & Context
- 小程序使用 Taro 4.x + React + TypeScript
- 数据源已切换为后端 API（useLocal: false）
- 后端服务运行在 4000 端口，数据库已配置
- 已配置微信小店插件
- 共 23 个页面，4 个 Tab 页

## Functional Requirements
- **FR-1**: 首页流程验证 - Banner 跳转、课程卡片跳转、讲师卡片跳转
- **FR-2**: 课程浏览流程 - 课程列表 → 课程详情 → 课时播放
- **FR-3**: 讲师浏览流程 - 讲师列表 → 讲师详情
- **FR-4**: 个人中心流程 - 各菜单项跳转、用户信息加载
- **FR-5**: 学习页面流程 - 学习列表 → 继续学习
- **FR-6**: 购买流程 - 课程详情页 → 微信小店、播放页 → 微信小店
- **FR-7**: API 接口验证 - 所有页面调用的接口均返回正确数据

## Non-Functional Requirements
- **NFR-1**: 所有页面的 API 调用均能正常返回（200 状态码）
- **NFR-2**: 页面跳转目标路径正确，无 404 页面
- **NFR-3**: 页面参数传递正确，详情页能获取到对应数据
- **NFR-4**: 错误处理机制完善，接口失败时有友好提示

## Constraints
- **Technical**: 
  - 仅验证代码层面的流程正确性
  - 不引入新依赖，仅修复现有问题
  - 使用后端 API 模式（useLocal: false）
- **Business**: 
  - 修复需遵循最小改动原则
  - 不改变现有业务逻辑
- **Dependencies**: 
  - 后端服务运行中
  - 数据库数据完整

## Assumptions
- 后端服务正常运行在 localhost:4000
- 数据库中有完整的测试数据
- 小程序服务层已正确配置 API 地址
- 页面代码结构与 AGENTS.md 规范一致

## Acceptance Criteria

### AC-1: 首页流程完整
- **Given**: 小程序启动，进入首页
- **When**: 点击 Banner、课程卡片、讲师卡片、查看全部按钮
- **Then**: 跳转到对应页面，页面能正确加载数据
- **Verification**: `programmatic`
- **Notes**: 验证跳转路径、参数传递、目标页面数据加载

### AC-2: 课程浏览流程完整
- **Given**: 用户在课程列表页
- **When**: 点击课程卡片进入详情，再点击立即学习进入播放页
- **Then**: 课程详情页显示正确课程信息，播放页显示正确课时内容
- **Verification**: `programmatic`
- **Notes**: 验证 courseId、lessonId 参数传递

### AC-3: 讲师流程完整
- **Given**: 用户在首页或讲师列表页
- **When**: 点击讲师卡片进入讲师详情页
- **Then**: 讲师详情页显示正确的讲师信息
- **Verification**: `programmatic`

### AC-4: 个人中心流程完整
- **Given**: 用户在个人中心页
- **When**: 点击各菜单项（我的课程、收藏、订单、设置等）
- **Then**: 跳转到对应页面，页面能正常加载数据
- **Verification**: `programmatic`
- **Notes**: 验证所有 12+ 个菜单项的跳转

### AC-5: 学习页面流程完整
- **Given**: 用户在学习 Tab 页
- **When**: 点击继续学习或课程卡片
- **Then**: 跳转到对应课程的播放页
- **Verification**: `programmatic`

### AC-6: 购买流程可触发
- **Given**: 用户在课程详情页或播放页（未购买状态）
- **When**: 点击购买按钮
- **Then**: 微信小店组件正确渲染，或跳转到对应页面
- **Verification**: `human-judgment`

### AC-7: 所有页面 API 调用正常
- **Given**: 后端服务正常运行
- **When**: 进入任意页面
- **Then**: 页面 API 调用返回 200，数据结构正确，无报错
- **Verification**: `programmatic`
